海洋环境保护科幻主题网页开发文档

一、项目头脑风暴全维度补充

（一）核心创意深化

以可交互3D地球为网页绝对核心，摒弃传统网页布局，打造沉浸式太空视角海洋环保科幻界面，将地球作为唯一视觉主体，承载所有海洋环保信息展示。打破平面限制，实现地球360°无死角拖动、旋转、缩放交互，每一处海洋污染、生态保护重点区域，都以科幻感标记点呈现，点击后触发全息悬浮窗，用视觉化内容传递海洋环保数据，让用户在科幻交互中感知海洋危机与保护意义。

（二）整体架构设计

1. 页面层级架构

  - 底层：深空渐变背景（藏青→深海蓝→浅蓝过渡），搭配微弱星点、星云动态效果，营造宇宙太空氛围，无多余装饰，突出地球主体。

  - 中层：3D可交互地球模型（核心载体），表面还原海洋、陆地轮廓，海洋区域做半透明波光动态效果，标注环保标记点（悬浮光点/全息图标）。

  - 上层：全局导航栏（极简科幻风，固定顶部，半透明隐藏式）、交互提示浮层、点击触发的全息悬浮信息窗、页面加载/切换动效层。

  - 弹窗层：详情信息全屏浮层，支持关闭、切换相邻标记点内容，不破坏整体页面沉浸感。

2. 功能模块架构

  - 3D地球交互模块（旋转、拖动、缩放、自动旋转待机）

  - 环保标记点触发模块（光点呼吸动画、点击高亮、定位联动）

  - 全息悬浮窗展示模块（图文/视频混排、渐入渐出、悬浮飘动）

  - 全局视觉动效模块（背景星云流动、海洋波光、标记点粒子特效）

  - 响应式适配模块（PC端完整交互，移动端简化交互+适配布局）

  - 页面引导模块（新手交互提示、主题标语、版权信息）

（三）视觉风格统一规范

1. 主色调体系

  - 主色：深海蓝（#0A192F）、海洋青（#115D8C）、浅滩蓝（#4FC3F7）

  - 辅助色：极光紫（#7B61FF）、生态绿（#00DFC0）、警示橙（#FF9800）（用于污染区域标记）

  - 中性色：深空黑（#000000）、月光白（#F5F5F5）、半透灰（rgba(255,255,255,0.1)）

2. 美术风格定位

  - 整体：科幻赛博海洋风，融合太空宇宙的浩瀚感与海洋的深邃感，兼具科技感与环保温情，无冗余元素，极简且高级。

  - 地球模型：低多边形+高清纹理结合，海洋部分做动态波光渲染，陆地做哑光处理，突出海洋占比，标记点采用全息投影式设计，无实体边框。

  - 文字：无衬线科幻字体（Inter、Roboto Condensed），标题字重偏粗带微弱发光效果，正文纤细清晰，文字颜色以浅蓝、白色为主，警示信息用橙红强调。

  - 动效：全页面采用流体动效，旋转顺滑、弹窗渐入、粒子消散，无突兀跳转，所有交互反馈延迟≤0.3s，营造沉浸式科幻体验。

3. 元素设计规范

  - 标记点：分三类——蓝色光点（生态良好海域）、橙色光点（污染预警海域）、绿色光点（环保保护示范区），均做呼吸缩放+粒子拖尾动画，悬浮于地球表面上方。

  - 悬浮窗：半透磨砂玻璃质感，无实体边框，边角做微圆弧，窗口带轻微上下漂浮动画，边框有微弱蓝光流光效果，贴合科幻全息概念。

  - 图标：线性科幻图标，线条纤细，带微弱发光，统一海洋/环保主题（水滴、浪花、鲸鱼、垃圾桶、检测仪器等）。

（四）交互逻辑补充

1. 地球基础交互

  - 鼠标拖动：地球360°水平+垂直旋转，旋转速度跟随鼠标拖动速度，松手后自然减速停止。

  - 鼠标滚轮：缩放地球（最小全局视角，最大可放大至单片海域细节），缩放至指定阈值时，对应区域标记点放大高亮。

  - 待机状态：无操作5s后，地球自动缓慢旋转，展示全球海域，标记点同步呼吸闪烁。

2. 标记点交互

  - 悬停：标记点放大，粒子拖尾变密集，弹出极简提示框（显示海域名称）。

  - 点击：地球自动旋转至该标记点居中位置，同步缩放至合适视角，标记点爆发粒子特效，随后弹出对应全息悬浮窗。

  - 关闭：点击悬浮窗关闭按钮，窗口消散成粒子，地球恢复自动旋转，标记点回归普通状态。

3. 悬浮窗交互

  - 窗口：轻微上下漂浮，支持鼠标拖动悬浮窗位置，不影响地球底层交互。

  - 内容：支持图文滑动、视频播放/暂停，窗口内有分页切换按钮，可查看该海域多组环保数据。

  - 联动：点击悬浮窗“相邻区域”按钮，地球自动旋转至下一个标记点，切换悬浮窗内容。

（五）内容板块规划

1. 首页核心内容

  - 主题标语：页面顶部极简半透明文字，“守护蔚蓝星球，共护海洋生态”，带微弱发光效果。

  - 交互提示：首次进入页面，弹出极简提示“拖动旋转地球，点击光点探索海洋秘密”，3s后自动消散。

  - 地球主体：全球3D地球，标注10-12个核心标记点，覆盖全球重点海洋环保区域。

2. 标记点详情内容（按区域分类）

  - 生态优质海域：展示海水水质数据、海洋生物多样性图片、珊瑚礁生态视频，搭配保护成果文字介绍。

  - 污染预警海域：展示塑料污染、石油污染、海水酸化实拍图/短视频，标注污染指数、污染源、危害分析。

  - 环保保护示范区：展示人工鱼礁、海洋垃圾清理、濒危动物保护实拍内容，搭配保护措施、公益行动呼吁文字。

3. 底部附加内容

  - 极简版权信息、海洋环保公益组织链接、环保倡议小标语，半透明隐藏，鼠标悬停显示。

（六）技术写法与选型

1. 前端技术栈

  - 核心框架：纯前端实现，无后端，采用HTML5+CSS3+JavaScript基础架构。

  - 3D地球实现：Three.js（核心库，实现3D模型加载、旋转、缩放、交互，轻量化配置，保证加载速度）。

  - 样式处理：Tailwind CSS（快速实现半透、磨砂玻璃、渐变效果）+ CSS3（动效、动画、响应式）。

  - 交互优化：GSAP（实现流畅动画、粒子效果、悬浮窗飘动），避免原生JS动画卡顿。

  - 媒体处理：懒加载图片、视频，压缩媒体文件体积，保证页面加载速度。

2. 代码写法规范

  - 模块化开发：将3D地球、标记点、悬浮窗、动效拆分为独立JS模块，便于维护和修改。

  - 性能优化：Three.js模型轻量化，减少面数，禁用多余光影效果；媒体文件懒加载，避免首次加载卡顿。

  - 响应式代码：采用rem+flex布局，针对PC、平板、移动端编写不同交互逻辑，移动端禁用复杂旋转，改为点击切换视角。

  - 注释规范：关键代码、交互逻辑、3D配置参数添加详细注释，便于后续修改和扩展。

3. 兼容性与性能

  - 兼容主流浏览器：Chrome、Firefox、Edge、Safari，针对低版本浏览器做降级处理（展示2D地球+静态信息）。

  - 性能优化：压缩代码、图片、视频，开启浏览器缓存，确保低配电脑也能流畅运行3D交互。

2. 样式文件（style.css）

  - interaction.js：交互逻辑，鼠标拖动/滚轮事件、标记点点击事件、悬浮窗显示隐藏、GSAP动效实现

（四）核心功能开发细则

1. 3D地球实现（earth.js）

  - 创建场景、相机、渲染器，设置深空背景色，添加环境光与点光源，营造科幻光影

  - 加载地球纹理（海洋+陆地高清纹理），创建球体模型，设置旋转速度、缩放范围（最小0.5倍，最大3倍）

  - 遍历标记点坐标数据，在地球对应位置创建光点模型，添加呼吸动画、粒子拖尾效果

  - 绑定鼠标事件，实现拖动旋转、滚轮缩放，监听交互状态，控制自动旋转待机

2. 标记点交互（interaction.js）

  - 获取标记点坐标，绑定鼠标点击/悬停事件，悬停时显示海域名称提示

  - 点击标记点，通过GSAP实现地球自动旋转居中、缩放，标记点粒子爆发特效

  - 从data.js读取对应数据，动态填充至悬浮窗，显示悬浮窗并添加漂浮动画

3. 悬浮窗功能

  - 实现磨砂玻璃样式，设置上下漂浮动画，鼠标可拖动调整位置

  - 支持视频播放、图片轮播，内容超出时添加滚动条，保持样式统一

  - 关闭按钮绑定事件，悬浮窗消散粒子动画，地球恢复自动旋转

4. 响应式适配

  - PC端：支持完整拖动、旋转、缩放、多细节交互

  - 平板端：禁用复杂旋转，改为点击左右按钮旋转，缩放范围缩小

  - 移动端：全屏展示2D简化地球，点击标记点直接弹出全屏详情页，保证操作便捷

（五）视觉与动效开发规范

## 新增：PostgreSQL 后端与实时搜索（2026-04-14）

- **目的**：提供一个简易的 API，将 `markersData` 中的 16 个监测点落地到本地 Postgres 中，同时为首页提供一个 `Ctrl / ⌘ + K` 即可唤起的搜索面板。
- **入口**：`server/` 目录包含 Express 代码、`scripts/schema.sql`（表结构）、`scripts/seed-markers.js`（导入 mock 数据）。
- **前端联动**：`index.html` 顶部新增「数据检索」按钮，同时增加 `window.__BLUE_AEGIS_API__` 配置项，默认指向 `http://localhost:4000`。

### 如何启动后端

1. 安装并启动 PostgreSQL，确保可以使用密码 `roots123` 连接（可根据个人环境调整）。
2. 进入 `server` 目录并安装依赖：
   ```bash
   cd server
   npm install
   ```
3. 复制 `.env.example` 为 `.env`，修改 `DATABASE_URL`（示例：`postgresql://postgres:roots123@localhost:5432/ocean_guardian`）。
4. 初始化数据库并导入 mock 数据：
   ```bash
   npm run seed
   ```
5. 启动接口服务（默认端口 `4000`）：
   ```bash
   npm run dev
   ```

### API 速览

| Method | Path | 说明 |
| --- | --- | --- |
| GET | `/api/markers` | 列出监测点，支持 `limit` / `offset` / `type` 参数 |
| GET | `/api/markers/:slug` | 返回单个监测点的所有字段 |
| GET | `/api/markers/search?q=keyword` | 按名称、标题、简介等字段模糊搜索 |

若部署到其他域名 / 端口，可在页面加载前通过 `window.__BLUE_AEGIS_API__ = 'https://your-host'` 覆盖默认值。
